# 📖 ノベルチェーン — サービス紹介資料

> **みんなで紡ぐ、物語の連鎖。**

---

## 1. サービス概要

### ノベルチェーンとは？

**ノベルチェーン**は、ユーザーが小説を投稿・共有し、他のユーザーが新しい章を提案して**共同で物語を作り上げていく**Webサービスです。

一人の作者が物語を始め、世界中の読者がその続きを提案する ── 物語が"チェーン"のように繋がっていく、新しい小説体験を提供します。

### コアコンセプト

| コンセプト | 説明 |
|---|---|
| 🖊️ **誰でも作家** | ログインするだけで小説を投稿し、みんなに読んでもらえる |
| 🔗 **物語の共同創作** | 他の人の小説に「こんな続きはどう？」と章を提案できる |
| ✅ **承認システム** | 提案された章は、作者の承認やユーザー投票で正式採用 |
| ❤️ **お気に入り** | 気になる小説をブックマークして追いかけられる |
| ⛓️ **ブロックチェーン連携** | Symbol独自トークンで投票権を管理する先進的な仕組み |

## 3. 主要機能の紹介

### 3.1 🏠 トップページ — 小説を見つける

トップページでは、すべての小説が**カードレイアウト**で一覧表示されます。

**カード構成:**

```
┌──────────────────────────────────────────┐
│ ┌─────┐  タイトル           📜縦  ❤️   │
│ │ 📕  │  作者名                         │
│ │表紙 │  説明テキスト...                │
│ │画像 │                                 │
│ └─────┘  ┌──────────────────┐           │
│          │ 最新章プレビュー │           │
│          └──────────────────┘           │
│              すべての章を読む →          │
└──────────────────────────────────────────┘
```

**主な機能:**

- 🔍 **タイトル検索** — 検索バーからリアルタイムに絞り込み（デバウンス付き）
- 📄 **ページネーション** — 20件ずつ表示、ページ番号で移動
- 📱 **レスポンシブ対応** — モバイル1列 → タブレット2列 → デスクトップ3列
- 📜 **縦読み対応** — 縦読み小説の最新章は縦書きでプレビュー表示
- ❤️ **お気に入り表示** — ログイン中はお気に入り済みの小説にハートマークが表示

---

### 3.2 📖 小説の作成と管理

#### 小説を作る

ログインユーザーなら誰でも小説を始められます。

| 入力項目 | 必須 | 説明 |
|---|---|---|
| タイトル | ✅ | 小説のタイトル（200文字以内） |
| 説明 | — | あらすじや概要 |
| 最初の章 | ✅ | 物語の始まりとなる文章 |
| 承認方法 | ✅ | 作者承認 / 投票承認から選択 |
| 読み方向 | ✅ | 横読み（デフォルト）/ 縦読み |

#### 小説を編集する

作者は後からタイトル・説明・承認方法・読み方向を変更でき、**カバー画像**のアップロードも可能です。

**カバー画像の特徴:**
- A5判比率（148×210）に自動リサイズ
- WebP形式に自動変換
- サムネイルも自動生成
- Cloudflare R2に保存

### 3.4 🔗 提案システム — 物語の共同創作

ノベルチェーンの最大の特徴が**提案システム**です。

#### 提案の流れ

```
  作者が小説を作成
        │
        ▼
  作者が「📢 公募」を開始
        │
        ▼
  読者が「次の章」を提案
        │
        ├─── 作者承認モード ──→ 作者が確認して承認
        │
        └─── 投票承認モード ──→ ユーザーが投票
                                    │
                                    ▼
                               作者が投票数を参考に承認
        │
        ▼
  提案が正式な章として追加！ 🎉
  （公募は自動終了・他の提案は削除）
```

#### 承認方法の比較

| | 🖊️ 作者承認 | 🗳️ 投票承認 |
|---|---|---|
| **決め方** | 作者が手動で判断 | 作者が投票数を参考に判断 |
| **投票機能** | なし | あり（ユーザーが投票可能） |
| **向いている** | 作者が物語を管理したい | みんなの意見で物語を作りたい |

#### 提案へのコメント

- 💬 各提案にコメントを投稿可能（ログイン不要）
- 小説の作者のコメントには**「作者」バッジ**が表示
- 作者はコメントを削除する権限を持つ

#### 提案管理ページ

自分が出した提案を一覧管理：
- **「提案中」タブ** — 承認待ちの提案（編集・取り消し可能）
- **「承認済み」タブ** — 正式に章になった提案

### 3.6 ⛓️ Symbolブロックチェーン連携

**先進的な機能**として、Symbolブロックチェーン（テストネット）と連携しています。

#### できること

| 機能 | 説明 |
|---|---|
| 🔗 **ウォレット接続** | プロフィールからSSS Extension経由でSymbolアドレスを紐付け |
| 🪙 **独自トークン発行** | 作者が自分のトークン（モザイク）を発行できる |
| 🗳️ **トークン投票** | トークン保有者だけが投票できる特別な投票モード |

#### トークン投票の仕組み

```
  作者がトークンを発行（SSS Extensionで署名）
        │
        ▼
  トークンをファンや読者に配布
        │
        ▼
  公募開始 → 提案が集まる
        │
        ▼
  トークン保有者がSSS署名で投票
  （ログイン不要・ウォレット署名で本人確認）
        │
        ▼
  作者が投票結果を参考に承認
```

**技術的なポイント:**
- Ed25519署名による本人確認
- Symbol REST APIでトークン保有を検証
- ログイン不要でブロックチェーンベースの投票が可能

## 5. 操作シナリオ

### シナリオ① 小説を読む（ログインなし）

```
1. トップページにアクセス
2. 🔍 検索バーでタイトルを検索、または一覧をスクロール
3. 気になる小説カードの「すべての章を読む →」をタップ
4. 小説詳細ページで全章を閲覧
```

### シナリオ② 小説を書く

```
1. ログインする
2. ボトムナビの「📖 マイ小説」をタップ
3. 「✏️ 執筆を始める」ボタンをタップ
4. タイトル・説明・最初の章・承認方法・読み方向を入力
5. 「作成」をタップ → 小説が公開！
6. 続きの章を書く場合は詳細ページから「章を追加」
```

### シナリオ③ 他の人の小説に続きを提案する

```
1. ログインする
2. トップページで「📢 公募中」の小説を見つける
3. 小説詳細ページを開く
4. 「📝 章を提案」ボタンをタップ
5. 次の章の内容を書いて提出
6. 作者の承認を待つ → 承認されると正式な章に！🎉
```

### シナリオ④ 提案を募集して共同創作する

```
1. 自分の小説の詳細ページを開く
2. 「📢 公募を開始する」をタップ → 提案受付開始
3. 読者からの提案が集まる
4. 提案を確認し、コメントでフィードバック
5. 気に入った提案を「✅ 承認」→ 正式な章に追加！
   （公募は自動終了、他の提案は削除）
6. 次の章を書くか、再び公募を開始して繰り返す
```

### シナリオ⑤ トークン投票を活用する

```
1. プロフィール画面でSSS Extensionを使ってウォレット接続
2. 「🪙 トークン発行」で独自トークンを作成
3. トークンをファンや読者に配布
4. 小説の承認方法を「投票承認」に設定
5. 公募を開始 → 提案が集まる
6. トークン保有者がSSS署名で投票
7. 投票結果を参考に承認 → みんなで作る物語の完成！
```

## 7. サービスの特徴・差別化ポイント

### ① 共同創作という新しい体験

従来の小説投稿サイトは「一人で書いて、みんなに読んでもらう」モデル。  
ノベルチェーンは**「みんなで物語を紡いでいく」**という新しい創作体験を提供します。

### ② 柔軟な承認システム

作者が物語の方向性をコントロールしたい場合は**作者承認**、  
読者の意見を取り入れたい場合は**投票承認**と、  
作品のスタイルに合わせて選択できます。

### ③ ブロックチェーンで実現するトークンエコノミー

Symbolブロックチェーンの独自トークンを活用し、  
**トークン保有者だけが投票できる**仕組みを実現。  
作者とファンの新しい関係性を構築します。

### ④ 縦書き対応で和の体験

日本の伝統的な**縦書き（縦読み）**に対応。  
小説の雰囲気に合わせて横書き・縦書きを選択でき、  
和風小説や時代小説に最適な読書体験を提供します。

### ⑤ モバイルファーストの設計

スマートフォンでの利用を前提とした設計で、  
通勤中や休憩時間にもサクッと小説を読んだり提案したりできます。

### ⑥ エッジコンピューティングで高速体験

Cloudflare Workers / Pages / D1 / R2を活用し、  
世界中のエッジから高速にコンテンツを配信。  
低レイテンシーで快適なユーザー体験を実現しています。
