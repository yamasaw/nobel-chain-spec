# Novel Chain — 機能仕様書

## 1. サービス概要

**ノベルチェーン** は、ユーザーが小説を投稿・共有し、他のユーザーが新しい章を提案して共同で物語を作り上げていくサービスです。

### コンセプト

- 誰でも小説を投稿し、みんなに読んでもらえる
- 他のユーザーの小説に「こんな続きはどう？」と章を提案できる
- 提案された章は、作者の承認 または ユーザーの投票 によって正式採用される
- お気に入り機能で気になる小説をブックマークできる

---

## 2. ユーザーアカウント

### 2.1 ログイン・登録

- メールアドレスまたはソーシャルアカウントでログイン・新規登録が可能
- 初回ログイン時にアカウントが自動作成される

### 2.2 プロフィール

プロフィールページでは以下の情報を確認・編集できます。

| 項目           | 説明                             | 編集           |
| -------------- | -------------------------------- | -------------- |
| ユーザー名     | サービス上の表示名               | ✅             |
| アバター画像   | プロフィール写真                 | ✅             |
| 自己紹介       | 自由記述（500文字以内）          | ✅             |
| メールアドレス | ログインに使用する               | ❌（表示のみ） |
| Symbolアドレス | 紐付けたブロックチェーンアドレス | ✅（SSS接続）  |
| 独自トークン   | 発行した独自トークン（モザイク） | ✅（発行操作） |

- 章や提案にはユーザー名とアバター画像が表示されます

### 2.3 ログインが必要な操作

以下の操作にはログインが必要です。

- 小説の作成・編集・削除
- タイトル画像のアップロード・削除
- 章の追加・提案
- 提案の承認・削除
- 投票・投票取り消し
- お気に入りの登録・解除
- プロフィールの編集
- プロフィール画面からのログアウト
- 提案管理の閲覧

> 小説の閲覧・章の閲覧・提案の閲覧はログインなしで可能です。

## 3. 小説

### 3.1 小説の作成

ログインユーザーは誰でも新しい小説を作成できます。作成時に入力する情報は以下の通りです。

| 入力項目 | 必須 | 説明                                                   |
| -------- | ---- | ------------------------------------------------------ |
| タイトル | ✅   | 小説のタイトル（200文字以内）                          |
| 説明     | —    | 小説の概要やあらすじ                                   |
| 最初の章 | ✅   | 物語の始まりとなる文章                                 |
| 承認方法 | ✅   | 提案の承認方法を選択（後述）                           |
| 必要票数 | ※    | 投票承認またはトークン投票を選んだ場合のみ（1〜100票） |
| 読み方向 | ✅   | 横読み（デフォルト）または縦読み                       |

### 3.2 小説の編集

小説の作者は以下の項目を後から変更できます。

- タイトル
- 説明
- 承認方法（作者承認 / 投票承認 / トークン投票の切り替え）
- 必要票数
- 読み方向（横読み ↔ 縦読みの切り替え）
- タイトル画像（後述）

### 3.2.1 タイトル画像

小説の編集画面から、タイトル用のカバー画像をアップロードできます。

- 画像は **A5判の比率（148×210）** に自動リサイズされます（フル画像: 592×840px）
- サムネイル（100×142px）も同時に生成されます
- 画像形式は WebP に自動変換されます
- 画像は Cloudflare R2 に保存されます
- 1枚のみ設定可能で、再アップロードすると上書きされます
- 画像を削除することもできます
- 画像は任意（nullable）であり、設定しなくても小説は正常に表示されます

#### 画像の表示

| 表示箇所         | 表示する画像 |
| ---------------- | ------------ |
| 小説詳細ページ   | カバー画像   |
| トップページ一覧 | サムネイル   |
| マイ小説一覧     | サムネイル   |
| お気に入り一覧   | サムネイル   |

### 3.3 小説の削除

小説の作者は小説を削除できます。削除すると、関連する章・提案・お気に入り・投票・カバー画像もすべて削除されます。

### 3.4 小説の閲覧

#### トップページ

- すべての小説が一覧表示されます（デフォルト20件ずつ）
- **タイトル検索**: 検索バーからタイトルの曖昧検索が可能です（デバウンス付き、400ms）
- **ページネーション**: 小説が複数ページにまたがる場合、ページ番号ナビゲーションで移動できます
  - 取得件数はAPIのlimitパラメータで任意に変更可能（デフォルト20件、最大100件）
- **カードレイアウト**: 各小説はNovelCardコンポーネントで表示されます
  - 左側に表紙画像（未設定時はプレースホルダーアイコン）
  - 右側にタイトル・作者名・説明・最新章プレビュー
  - 縦読み小説にはタイトル横に「📜 縦」バッジが表示されます
  - ログイン中はお気に入り済みの小説に ❤️ アイコンが表示されます
  - 縦読み小説の最新章プレビューは縦書きで表示されます
  - カード下部に「すべての章を読む →」リンクで詳細ページへ遷移
- **レスポンシブ対応**: flexbox wrapによるグリッドレイアウト
  - モバイル: 1列、タブレット(sm): 2列、デスクトップ(lg): 3列

#### 小説詳細ページ

- 小説のタイトル・説明・作成日が表示されます
- 全章が時系列順に表示されます（各章には作成者の名前とアバターが表示）
- 投票承認モードの小説には「🗳️ 投票承認（N票で可決）」バッジが表示されます
- 公募中の小説には「📢 公募中」バッジが表示されます
- 読み方向バッジが表示されます（「📜 縦読み」または「📖 横読み」）
- 縦読みの小説は、章の本文が **縦書き（右から左、上から下）** でレイアウトされます
- ページ下部に提案一覧が表示されます

#### マイ小説

- ログインユーザーが作成した小説の一覧を表示
- 各小説から編集・章追加画面へ遷移可能

## 4. 章

### 4.1 章の追加

小説の作者は、詳細ページから直接新しい章を追加できます。追加された章は最新章の後に続きます。

> ⚠️ 公募中（提案を募集中）の小説には、作者であっても直接章を追加することはできません。章を追加するには、先に公募を終了する必要があります。

### 4.2 章の表示

- 各章には **第N章** の番号が付けられます
- 章の作成者のユーザー名・アバター画像・作成日が表示されます
- 提案から承認された章の場合、提案者の情報が作成者として表示されます

## 5. 提案システム

### 5.1 提案とは

「提案」とは、他のユーザー（または作者自身）が小説に対して「次の章の案」を提出する機能です。提案はすぐに小説に反映されるわけではなく、承認されて初めて正式な章として追加されます。

### 5.2 提案の作成

- **すべてのログインユーザー**が、**公募中の小説**に章を提案できます
- 公募中でない小説には提案できません
- 小説の作者自身も自分の小説に提案を出すことができます
- 提案時に入力するのは **章の内容** のみです

### 5.3 提案の承認方法

小説には3つの承認方法があり、作成時に選択します（後から変更も可能）。

#### ① 作者による承認（デフォルト）

- 小説の作者が提案を確認し、「承認する」ボタンを押すと正式な章として追加されます
- 承認するかどうかは作者の判断に委ねられます

#### ② 投票による承認

- ログインユーザーが提案に対して投票できます
- 投票数が小説に設定された **必要票数** に達すると、自動的に正式な章として追加されます
- 作者は投票数に関係なく、いつでも手動で承認することもできます

#### ③ トークン投票による承認

- 作者が発行した **独自トークン（モザイク）の保有者のみ** が投票できます
- **ログイン不要** — SSS Extension によるウォレット署名で本人確認を行います
- 投票数が必要票数に達すると、自動的に正式な章として追加されます
- 作者は投票数に関係なく、いつでも手動で承認することもできます
- この承認方法を選択するには、作者が事前にトークンを発行している必要があります

### 5.4 承認方法の比較

| 項目           | 作者承認               | 投票承認                        | トークン投票                             |
| -------------- | ---------------------- | ------------------------------- | ---------------------------------------- |
| 誰が決めるか   | 作者が手動で判断       | ユーザー全体の投票で自動決定    | トークン保有者の投票で自動決定           |
| 投票できる人   | —                      | ログインユーザー全員            | 作者のトークン保有者のみ（ログイン不要） |
| 作者の手動承認 | ✅ これが唯一の方法    | ✅ いつでも可能（上書き承認）   | ✅ いつでも可能（上書き承認）            |
| 自動承認       | ❌ なし                | ✅ 必要票数に達すると自動で採用 | ✅ 必要票数に達すると自動で採用          |
| 必要なもの     | —                      | —                               | SSS Extension + トークン保有             |
| 向いている小説 | 作者が物語を管理したい | みんなで自由に物語を作りたい    | トークンコミュニティで物語を作りたい     |

### 5.5 承認されるとどうなるか

1. 提案の内容が正式な章として小説に追加されます
2. 章の作成者には **提案を書いた人** の名前が表示されます
3. 提案のステータスが「承認済み」に変わります
4. 小説の **公募が自動的に終了** します
5. 承認された提案以外の **未承認の提案はすべて削除** されます

### 5.6 提案の削除

- 提案を出した本人のみが削除できます
- まだ承認されていない提案だけが削除の対象です

### 5.7 提案の編集

- 提案を出した **本人のみ** が編集できます
- まだ承認されていない（pending 状態の）提案だけが編集の対象です
- 編集できるのは **章の内容** のみです
- 提案管理ページから編集できます

### 5.8 提案管理ページ

自分が提案した章をまとめて管理できるページです。

- **「提案中」タブ**: まだ承認されていない提案の一覧。取り消し（削除）が可能
- **「承認済み」タブ**: 承認されて正式な章になった提案の一覧

各提案から対象の小説ページへのリンクもあります。

### 5.9 公募（提案の募集）

「公募」とは、小説の作者が他のユーザーからの提案を受け付ける状態のことです。

#### 公募の開始・終了

- 小説の作者は、小説詳細ページから公募の **オン／オフ** を切り替えることができます
- 公募がオンの間は、小説に「📢 公募中」バッジが表示されます
- 公募がオフの小説には提案を出すことができません
- デフォルトでは公募はオフです

#### 公募中の制限

- 公募中の小説には、作者であっても直接章を追加することはできません
- 章を追加するには、先に公募を終了する必要があります

#### 公募の自動終了

- 提案が承認（作者による手動承認、または投票による自動承認）されると、公募は自動的に終了します
- 承認された提案以外の未承認の提案はすべて削除されます

#### 公募の手動終了

- 作者は公募をオフに切り替えることで手動で終了できます
- 公募を終了すると、**未承認の提案がすべて削除される** 旨の確認ダイアログが表示されます
- 確認後、未承認の提案がすべて削除され、公募が終了します

### 5.10 提案へのコメント

提案に対して意見や感想をコメントできる機能です。

#### コメントの投稿

- **ログイン不要**で、誰でもコメントを投稿できます
- 未ログインの場合、コメント投稿時には **名前**（1〜100文字）と **内容**（1〜1000文字）を入力します
- **ログインユーザー**がコメントした場合は、名前の入力は不要で、**ユーザー名が自動的に使用**されます
- ログインユーザーのコメントには **user_id** が保存されます
- まだ承認されていない（pending 状態の）提案にのみコメントできます

#### コメントの表示

- 小説詳細ページの各提案カードに 💬 ボタンが表示されます
- ボタンをクリックするとコメント一覧が展開されます
- **小説の作者**によるコメントには「作者」バッジが表示されます（user_id で判定、名前の偽装を防止）
- 作者のコメントは背景色で視覚的に区別されます

#### コメントの削除

- コメントを削除できるのは **小説の作者のみ** です
- コメント投稿者自身による削除はできません

#### コメントの自動削除

- いずれかの提案が承認されると、**その小説に紐づくすべてのコメントが自動的に削除** されます
- 公募を手動で終了した場合も、未承認の提案と一緒にコメントがすべて削除されます

## 6. 投票

### 6.1 投票の仕組み

投票には2つのモードがあり、小説の **承認方法** の設定によって自動的に決まります。

| 小説の承認方法   | 使用される投票モード |
| ---------------- | -------------------- |
| 作者承認         | 通常投票（※）        |
| 投票承認         | 通常投票             |
| トークン投票承認 | トークン投票         |

> ※ 作者承認の小説でも投票は可能ですが、票数に関わらず自動承認は行われません。

#### 通常投票（従来方式）

- ログインユーザーは、まだ承認されていない提案に対して **👍 投票する** ことができます
- 1人のユーザーが同じ提案に投票できるのは **1回** だけです
- 投票した後でも **投票を取り消す** ことができます

#### トークン投票（ブロックチェーン方式）

- 承認方法が「トークン投票」に設定された小説でのみ使用されます
- **トークン保有者のみ**が投票できます（通常のログインユーザーは投票不可）
- **ログイン不要**で、SSS Extension によるウォレット署名で本人確認を行います
- 投票時に SSS Extension でダミーの TransferTransaction に署名し、バックエンドで以下を検証します:
  1. Ed25519 署名の正当性（秘密鍵の所有証明）
  2. 公開鍵とアドレスの一致（Symbol REST API で検証）
  3. 作者のトークン（モザイク）の保有確認
- 1つの Symbol アドレスが同じ提案に投票できるのは **1回** だけです
- 投票した後でも **投票を取り消す** ことができます

### 6.2 投票の表示

- 各提案カードに現在の投票数が「🗳️ N票」形式で表示されます
- 投票承認・トークン投票承認の小説では「🗳️ N票 / M票」（N＝現在の票数、M＝必要票数）と表示されます
- 通常投票: 自分が投票済みの提案には「👍 投票済み」ボタンが表示されます
- トークン投票: SSS Extension 接続時、投票済みの提案には「🪙 投票済み」ボタンが表示されます
- トークン投票承認の小説には「🪙 トークン投票承認」バッジが表示されます
- トークン投票が必要な小説で SSS Extension が未インストールの場合、インストール案内が表示されます
- SSS Extension が接続済みでもトークンを保有していない場合、投票ボタンは無効になりません（バックエンドで検証）

### 6.3 自動承認

投票承認モードの小説では、提案の投票数が必要票数に達した瞬間に、自動的に以下が行われます。

1. 提案が正式な章として小説に追加される
2. ページが自動更新され、最新の状態が表示される

> 承認方法が「作者承認」の小説でも通常投票はできますが、投票数に関わらず自動承認は行われません。トークン投票承認の小説では通常投票はブロックされます。

## 7. お気に入り

### 7.1 お気に入りの登録・解除

- 小説詳細ページで ❤️ ボタンを押すとお気に入りに追加されます
- もう一度押すと解除されます
- 同じ小説を重複してお気に入りに登録することはできません

### 7.2 お気に入り一覧

- ヘッダーの「お気に入り」リンクからお気に入り一覧ページへ遷移できます
- お気に入りした小説が最新章プレビュー付きで表示されます
- カードを展開して詳細を確認し、「すべての章を読む」から小説詳細ページへ遷移できます
- 「お気に入りを解除」ボタンで確認ダイアログが表示され、解除できます

### 7.3 トップページでの表示

- トップページの小説一覧で、お気に入り済みの小説にはタイトル横に ❤️ アイコンが表示されます

## 8. Symbolブロックチェーン連携

### 8.1 概要

Symbol ブロックチェーン（テストネット）と連携し、独自トークン（モザイク）の発行・投票制限機能を提供します。ブラウザ拡張機能「SSS Extension」を介してウォレット操作を行います。

### 8.2 ウォレット接続（アドレス紐付け）

- プロフィール画面から SSS Extension を通じて Symbol アドレスを紐付けます
- SSS Extension が未インストールの場合、インストール案内が表示されます
- 接続後、アドレスはユーザーアカウントに保存されます
- 紐付け解除も可能です

### 8.3 独自トークン（モザイク）発行

- ウォレット接続済みのユーザーは、プロフィール画面からトークンを発行できます
- トークンのメタデータには `service=novel-chain` と `author={userId}` が設定されます
- 発行パラメータ:
  - 供給量: 任意（1〜999,999,999）
  - 譲渡可能 / 制限付き: 選択可能
  - 有効期限: 選択可能
- 発行済みの場合は追加発行（供給量の増加）が可能です
- 発行はすべて SSS Extension の署名確認を経て Symbol テストネットに送信されます

### 8.4 トークン投票

- 承認方法に「トークン投票」を選択した小説では、トークン保有者のみが投票できます
- この承認方法を選択するには、作者がウォレット接続済みかつトークン発行済みである必要があります
- 投票フローの詳細は `6.1 トークン投票（ブロックチェーン方式）` を参照してください
- 承認方法が「作者承認」または「投票承認」の小説では従来通りの通常投票が使用されます

### 8.5 技術仕様

| 項目                 | 値                                              |
| -------------------- | ----------------------------------------------- |
| ネットワーク         | Symbol テストネット                             |
| ノードURL            | `https://sym-test-01.opening-line.jp:3001`      |
| ウォレット連携       | SSS Extension（Chrome拡張）                     |
| フロントエンド SDK   | symbol-sdk@2.0.0 + rxjs（動的 import、SPA専用） |
| バックエンド署名検証 | crypto.subtle (Ed25519) — symbol-sdk 不使用     |
| トークン保有確認     | Symbol REST API (`GET /accounts/{address}`)     |
| 環境変数             | `SYMBOL_NODE_URL`（オプション、デフォルトあり） |
| フロントエンド配信   | Cloudflare Pages（SPA モード / adapter-static） |

## 9. 画面一覧
